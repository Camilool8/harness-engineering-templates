## Progress tracking (Linear)

Work items live in **Linear**, accessed via the Linear MCP.

- **At the start of a task**, read the relevant Linear issue in full —
  description, acceptance criteria, sub-issues and comments — before touching
  code.
- **As you work**, move the issue through workflow states so they reflect
  reality: set it `In Progress` when you start, comment material updates and
  blockers, and reference the issue identifier (e.g. `ENG-123`) on branches and
  PRs.
- **Never set an issue to `Done`** without verified evidence the acceptance
  criteria are met. If a PR is open but unmerged, the issue stays `In
  Review`, not `Done`. See the `tracking-progress-in-linear` skill.
