## Progress tracking (GitHub Issues)

Work items live in **GitHub Issues**, accessed via the GitHub MCP (or `gh` CLI).

- **At the start of a task**, read the relevant issue in full — description,
  acceptance criteria, and existing comments — before touching code.
- **As you work**, comment material progress on the issue: decisions made,
  blockers hit, what is done. Reference issues from commits and PRs (`#123`) so
  the trail is linked.
- **Never close an issue without verified evidence.** Closing happens only after
  the acceptance criteria are demonstrably met (tests pass, behavior confirmed).
  Prefer letting the merged PR close the issue. If unsure, leave it open and say
  so. See the `tracking-progress-in-github` skill.
